# atradio.fm Rust SDK — design

> Status: proposal · Target crate: `atradio-sdk` · Built on [jacquard](https://crates.io/crates/jacquard) 0.12

## Motivation

The `atradio` CLI (`cli/`) already contains a complete atradio.fm client built on
jacquard — it's just **fused into the binary** and not reusable. Anyone else who
wants to talk to atradio.fm from Rust (a bot, an alternate player, an indexer,
integration tests) has to copy code out of the CLI.

This document designs `atradio-sdk`: the **official Rust SDK for atradio.fm**,
extracted from and modeled on the pieces the CLI already proves in production,
with the ergonomic shape of Bluesky's [`@atproto/api`](https://github.com/bluesky-social/atproto/blob/main/packages/api/README.md).

**North star:** the CLI is refactored to depend on `atradio-sdk` — it becomes the
SDK's first-party consumer (the way `bsky-app` dogfoods `@atproto/api`). Nothing
in the CLI should talk to jacquard or the AppView directly once the SDK exists;
it goes through the SDK like any third-party client would. If the CLI needs
something the SDK can't express, that's an SDK gap to fill, not a reason to reach
around it.

## What already exists inline (the extraction map)

All three layers an SDK needs live in `cli/src/` today:

| Layer                     | Lives in today                             | Moves to (SDK)        |
| ------------------------- | ------------------------------------------ | --------------------- |
| Generated lexicon types   | `cli/src/fm_atradio/` + `builder_types.rs` | `src/lexicons/`       |
| Auth + writes (`Atproto`) | `cli/src/atproto/mod.rs`                   | `src/{agent,auth}.rs` |
| Public reads (`AppView`)  | `cli/src/appview/`                         | `src/appview.rs`      |

- **Generated lexicon types** — typed `fm.atradio.*` records, builders, and
  `Collection` impls, generated by `jacquard-lexgen` from
  `packages/lexicons/lexicons/atradio/*.json`.
- **Auth + writes** — app-password + OAuth login, `create`/`put`/`delete`
  records, service-auth minting, and the refresh-serialization mutex.
- **Public reads** — unauthenticated XRPC (`getRecentStations`, `getFavorites`,
  `getComments`, …).

Because the generated types move **byte-for-byte**, the SDK compiles the moment
the files land; the work is repackaging, not rewriting.

## Design goals

1. **CLI-first, but general.** The CLI's every need is satisfiable through the
   public SDK surface. No `pub(crate)` back doors.
2. **Mirror `@atproto/api` ergonomics** — an `Agent`, typed namespace accessors,
   and high-level convenience methods — expressed idiomatically in Rust.
3. **Thin over jacquard.** The SDK adds atradio semantics; it does not re-wrap
   every jacquard primitive. jacquard's `Agent`, `Collection`, and XRPC types
   remain reachable for escape hatches.
4. **Library-grade errors.** A typed `SdkError` (thiserror), never `anyhow`.
5. **Feature-gated** so a read-only bot doesn't pull the OAuth/loopback stack.

## Mapping to `@atproto/api`

| `@atproto/api` (TypeScript)                | `atradio-sdk` (Rust)                                      |
| ------------------------------------------ | --------------------------------------------------------- |
| `new Agent(session)`                       | `AtradioAgent::new(session)`                              |
| `session.login({ identifier, password })`  | `AtradioAgent::login_password(id, pw)`                    |
| OAuth via `@atproto/oauth-client-*`        | `AtradioAgent::login_oauth(hint)` (loopback)              |
| `agent.com.atproto.repo.createRecord(...)` | jacquard `agent.create_record(record)` (escape hatch)     |
| `agent.app.bsky.feed.post.create(...)`     | `agent.station().create(record)` (typed namespace)        |
| `agent.post()` / `.like()` / `.follow()`   | `agent.comment()` / `.favorite()` / `.set_play_status()`  |
| `agent.getProfile()` / `getTimeline()`     | `agent.appview().recent_stations()` / `.favorites(actor)` |
| `new RichText().detectFacets(agent)`       | `atradio_sdk::facets::detect_mentions(text)`              |
| cursor pagination on list methods          | `Page<T> { items, cursor }` + `into_stream()`             |

## Crate layout

The repo has no root `Cargo.toml` today (only `cli/Cargo.toml`). Introduce a
workspace:

```
Cargo.toml                      # [workspace] members = ["crates/*", "cli"]
crates/
  atradio-sdk/
    Cargo.toml
    README.md                   # quickstart (this doc's snippets)
    src/
      lib.rs                    # re-exports; crate docs
      lexicons/                 # ← generated, verbatim from cli/src/fm_atradio/
        mod.rs                  #   fm.atradio.* records + defs
        builder_types.rs        #   Set/Unset builder markers
      agent.rs                  # AtradioAgent — the star
      auth.rs                   # login/resume, session store, atradio_scopes()
      appview.rs                # public read client + pagination
      namespaces.rs             # typed accessors: .station() .favorite() .comment()
      facets.rs                 # mention/gif facet helpers
      error.rs                  # SdkError
cli/                            # depends on atradio-sdk; loses ~1,500 lines
```

**One crate, not two** (decided): generated lexicons live as a `lexicons` module
inside the SDK, exactly as `@atproto/api` ships its generated `client/` inside
the one package. The convenience layer is tightly coupled to the generated
types, and consumers want a single dependency. If a future pure-types consumer
(a standalone AppView/indexer) appears, `lexicons` can be split out then without
changing the SDK's public surface.

## Public API surface

### Construction & auth

```rust
use atradio_sdk::{AtradioAgent, AppViewUrl};

// Fresh login, persisted to a session file (jacquard FileAuthStore).
let agent = AtradioAgent::builder()
    .session_store("~/.config/atradio/session.json")
    .appview("https://api.atradio.fm")            // optional; sensible default
    .build()?;

agent.login_password("alice.bsky.social", &app_pw).await?;   // app password
// — or —
agent.login_oauth(Some("alice.bsky.social")).await?;         // browser + loopback

// Subsequent runs: resume whatever session was persisted (password OR oauth).
let agent = AtradioAgent::resume("~/.config/atradio/session.json").await?;

agent.is_logged_in();            // bool
agent.profile();                 // Option<Profile { did, handle, display_name, pds, method }>
agent.logout();
```

`login_oauth` / `resume` collapse today's duplicated `is_oauth()` branch (repeated
in every write method of `cli/src/atproto/mod.rs`) into one internal `write()`
path. The **auth mutex is load-bearing and moves into the SDK**: atproto rotates
the refresh token on every refresh, so concurrent refreshes reuse a token and the
PDS revokes the session. The SDK serializes authenticated ops behind one
`Arc<Mutex<()>>`, held across each op — same invariant the CLI relies on today
when the daemon writes play-status while the Connect loop mints service auth.

### Writes — convenience ("app") methods

The high-level verbs, mirroring `agent.post()` / `.like()`:

```rust
let uri = agent.favorite(&station).await?;                 // fm.atradio.favorite
agent.comment(&station, "great stream 🎶").await?;         // fm.atradio.comment
agent.create_station(&StationDraft {                       // fm.atradio.station
    name: "KEXP".into(),
    stream_url: "https://kexp.streamguys1.com/kexp160.aac".into(),
    genre: Some("eclectic".into()),
    ..Default::default()
}).await?;
agent.set_play_status(&station).await?;                    // put rkey=self
agent.delete_play_status().await?;
agent.put_audio_settings(&dsp).await?;                     // fm.atradio.audio.settings
let dsp = agent.get_audio_settings().await?;               // Option<AudioSettings>

// atradio Connect: mint a service-auth JWT for the AppView hub.
let jwt = agent.mint_service_auth(aud, lxm).await?;
```

### Writes — typed namespace accessors (escape hatch)

For records the convenience layer doesn't cover, or to set fields the verbs
don't expose:

```rust
let record = agent.station().draft()          // returns the generated builder
    .name("KEXP")
    .stream_url(url)
    .genre("eclectic")
    .build();
let out = agent.station().create(record).await?;   // -> CreateRecordOutput { uri, cid }
agent.favorite_ns().delete(rkey).await?;
```

These wrap jacquard's `agent.create_record` / `put_record` / `delete_record`,
which are already generic over any `Collection` — so the generated
`fm.atradio.*` types plug straight in.

### Reads — the AppView client

Lifted almost verbatim from `cli/src/appview/`; add cursor pagination (current
CLI signatures take only `limit`):

```rust
let av = agent.appview();                       // or atradio_sdk::AppView::new(url) standalone

let recent  = av.recent_stations(50).await?;                   // Vec<StationView>
let popular = av.popular_stations(50).await?;
let global  = av.global_recently_played(50).await?;

let page = av.favorites("alice.bsky.social").limit(30).await?; // Page<StationView>
let next = av.favorites("alice.bsky.social").cursor(page.cursor).await?;

// stream every page lazily:
use futures::TryStreamExt;
let all: Vec<_> = av.comments(&station_id).into_stream().try_collect().await?;

let counts = av.listener_counts(&station_ids).await?;          // Vec<ListenerCount>
let notifs = av.notifications("alice.bsky.social", 50).await?;
let unread = av.update_seen("alice.bsky.social").await?;       // POST, returns u32
```

`AppView` needs no auth, so `atradio_sdk::AppView::new(url)` is usable on its own
(a public discovery bot needs nothing else). `AtradioAgent` owns one internally
and returns it via `agent.appview()`.

### Facets

Comments carry `Mention` / `Gif` facets (`fm.atradio.comment`). A small helper
mirrors `RichText.detectFacets`:

```rust
use atradio_sdk::facets;
let mentions = facets::detect_mentions(&agent, "shout out @bob.bsky.social").await?;
agent.comment_with_facets(&station, text, mentions, gif).await?;
```

Grapheme-aware length + UTF-8/UTF-16 handling reuse the same
`unicode_segmentation` dependency the generated code already pulls in.

### Errors

```rust
#[derive(thiserror::Error, Debug)]
pub enum SdkError {
    #[error("not signed in")]                       NotAuthenticated,
    #[error("session expired — re-authenticate")]   SessionExpired,
    #[error("record not found")]                    RecordNotFound,
    #[error("appview {nsid} -> {status}: {body}")]  AppView { nsid: String, status: u16, body: String },
    #[error(transparent)]                           Jacquard(#[from] jacquard::Error),
    #[error(transparent)]                           Http(#[from] reqwest::Error),
    #[error(transparent)]                           Json(#[from] serde_json::Error),
}
pub type Result<T> = core::result::Result<T, SdkError>;
```

The CLI's current `msg.contains("RecordNotFound")` string-matching for missing
singletons (`get_audio_settings`) becomes a typed `SdkError::RecordNotFound`,
which getters map to `Ok(None)`.

## Feature flags

Mirror jacquard's, so consumers pull only what they use:

```toml
[features]
default = ["oauth", "dns", "appview"]
oauth   = []   # jacquard oauth + loopback (browser login)
dns     = []   # jacquard DNS handle resolver
appview = []   # reqwest-based public read client (usable without auth)
```

A read-only bot: `atradio-sdk = { version = "…", default-features = false, features = ["appview"] }`.

## Codegen story

`scripts/gen-lexicons.sh` keeps generating from
`packages/lexicons/lexicons/atradio/*.json` via `jacquard-codegen`; only its
**output directory** changes to `crates/atradio-sdk/src/lexicons/`. The generated
files stay committed (so `cargo install` / `cargo add` work without `protoc` or
`jacquard-lexgen` installed), exactly as today.

```bash
cargo install jacquard-lexgen        # provides jacquard-codegen
bash scripts/gen-lexicons.sh         # regenerates crates/atradio-sdk/src/lexicons/
```

## Migration path (proves the SDK)

Each step compiles and ships independently:

1. **Land the workspace + `crates/atradio-sdk`** with `lexicons/` = today's
   generated code, moved verbatim. Add `error.rs`, `appview.rs` (lifted from
   `cli/src/appview/`), and `agent.rs`/`auth.rs` (lifted from
   `cli/src/atproto/mod.rs`, with the OAuth/credential fork collapsed and
   `anyhow` replaced by `SdkError`).
2. **Repoint codegen** output to the SDK.
3. **Rewire the CLI**: replace `crate::atproto::Atproto` → `atradio_sdk::AtradioAgent`,
   `crate::appview::AppView` → `atradio_sdk::AppView`, `crate::fm_atradio::*` →
   `atradio_sdk::lexicons::*`. Delete the moved files. The CLI shrinks ~1,500
   lines and becomes the SDK's first consumer.
4. **Publish `atradio-sdk`** to crates.io. The `atradio` bin crate (v0.5.1) keeps
   its name and now lists `atradio-sdk` as a dependency.

The CLI's `remote.rs`, `grpc/`, `tui/`, `player/`, `mdns.rs`, `mpris.rs` stay in
the CLI — they're player/UI/transport concerns, not atradio-protocol concerns,
so they are **not** SDK surface. If any of them needs protocol data, it asks the
SDK.

## Open questions

- **AppView base URL default.** Hard-code `https://api.atradio.fm` as the default,
  overridable via the builder? (Leaning yes — matches how the CLI ships a default.)
- **`Profile` ownership.** The CLI's `Profile` (did/handle/display_name/pds/method)
  is really SDK-level identity state; move it into the SDK and re-export, with the
  CLI keeping only presentation concerns.
- **`StationDraft` / `StationInfo` home.** The wire `StationInfo`
  (`cli/src/appview/models.rs`) and the generated lexicon `StationInfo` are two
  types today, bridged by `gen_station()`. The SDK should expose **one** and keep
  the conversion internal.
- **Service-auth as first-class?** `mint_service_auth` is atradio-Connect-specific.
  Keep it on `AtradioAgent`, or move it behind a `connect` feature so pure-data
  consumers don't see it?
